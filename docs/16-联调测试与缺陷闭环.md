## 1. 联调范围

本轮从成功标准、前序开发记录、前序审计和任务板提取以下已承诺场景，不新增范围外测试：

### 1.1 用户端主流程

1. 切换体验账号并保持 Session。
2. 查看用户首页、座位列表和座位详情。
3. 选择日期和时段并提交预约。
4. 在“我的预约”查看刚创建的记录。
5. 合法取消预约并释放时段。

### 1.2 管理端主流程

1. 使用预置管理员登录并取得 AdminCookie。
2. 未登录访问后台时跳转登录页。
3. 按日期和状态筛选预约。
4. 管理员取消有效预约并保持筛选条件。
5. 新增、编辑、启用和停用座位。
6. 查看五项实时统计并退出登录。

### 1.3 状态回流和关键规则

1. 管理端取消后，用户端重新查询显示已取消。
2. 取消后座位时段和用户时段可重新预约。
3. SeatConflict 和 UserConflict 两类冲突均阻止重复预约。
4. 停用座位不可创建新预约，但已有预约不被删除。
5. 越权、开始后、重复和并发取消均被拒绝并显示安全提示。
6. U-P05、A-P02、A-P04 使用统一的 Reserved、Cancelled、Completed 口径。

### 1.4 体验和兼容性

1. 用户端在 390px 手机竖屏无溢出，主要操作可触达。
2. 管理端在 1366px 桌面视口保持侧边栏、筛选区、表格和操作区清楚。
3. Edge 和 Chrome 各完成一次主流程检查。
4. 空座位、空预约、空筛选和空统计有明确呈现。
