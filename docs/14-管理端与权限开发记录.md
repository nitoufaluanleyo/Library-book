## 1. 已确认的管理端口径

前序文档已经明确管理端不是审核流，而是登录保护下的状态维护闭环：

```text
A-P01 管理员登录
  -> A-P04 查看统计
  -> A-P02 按日期/展示状态筛选预约
  -> A-P02 管理员取消尚未结束的 Reserved 预约
  -> A-P04 重新查询统计
  -> A-P03 新增、编辑、启停座位
  -> A-P01 退出
```

权限边界已经明确：

- A-P01 允许匿名访问，登录成功签发 `AdminCookie`。
- A-P02、A-P03、A-P04 必须要求 `AdminCookie`。
- 管理员取消是状态维护，不新增审核状态；写回 `StoredStatus = Cancelled`、`CancelledBy = Admin` 和 `CancelledAt`。
- 用户端通过重新查库看到预约已取消、时段释放和统计变化。
- A-P02 首版只展示日期和状态筛选；区域和用户条件保留在 Service/DataAccess，不擅自新增页面控件。

本轮未进入实现，上述内容只作为后续 T15 的既有基线，不是新方案。
