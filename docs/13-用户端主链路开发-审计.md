## C. 必改项

1. **取得可运行工程前提。** 先完成开发前 P0 闭环和阶段 12 骨架验收，至少具备 Solution、Web 项目、build、run、Migration、LocalDB 和最小种子数据。
2. **重新开始阶段 13 的首轮任务。** 按任务板选择 T13-01 至 T13-03，完成体验用户 Session、首页真实查库和座位列表区域筛选，并按真实状态流转任务卡。
3. **完成用户浏览链路。** 后续完成 T13-04 至 T13-06，使 U-P01 -> U-P02 -> U-P03 可操作并通过数据隔离验证。
4. **完成预约闭环后再申请进入管理端。** 落地 U-P04、U-P05、真实写库、真实回查、表单验证、冲突校验和取消规则；至少演示一次预约成功并在“我的预约”看到结果。
5. **保留可复查证据。** 每轮记录 build 结果、实际路由、数据库读写结果、Session 用户、任务卡验收动作和失败场景，不用静态原型替代业务验收。

## D. 建议优化项

1. 首轮开发建议控制为 T13-01、T13-02、T13-03，先打通身份和查库，再推进详情与预约，便于定位问题。
2. Session 验收至少覆盖切换、刷新保持、非法用户回退和不同用户数据隔离。
3. 表单验证同时保留 ViewModel 的输入校验和 Service 的业务校验，Controller 只负责接收、调用和返回结果。
4. 真实读写验收记录建议包含操作前后数据库记录数量或关键字段，避免只凭页面提示判断成功。
5. README 当前状态准确；后续仅按真实实现增量勾选，不提前填写演示账号或可运行命令。
