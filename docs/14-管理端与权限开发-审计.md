## C. 必改项

1. **先满足管理端开工前提。** 恢复并完成阶段 12 工程骨架，随后完成用户端浏览和预约闭环，取得用户端阶段放行结论。
2. **先实现权限最小闭环。** 按 T15-01、T15-02 完成管理员哈希登录、AdminCookie、后台授权保护和退出，并验证未登录跳转和身份隔离。
3. **实现真实预约管理。** 按 T15-03、T15-04 从数据库查询、筛选和取消预约，状态写回必须沿用 `Reserved`、`Cancelled`、`CancelledBy = Admin` 和 `CancelledAt`。
4. **实现真实座位管理。** 按 T15-05、T15-06 完成 A-P03 同页新增、编辑、启停和编号唯一校验，不物理删除已有预约关联座位。
5. **实现统计与用户端回流。** A-P04 必须从数据库实时计算既定统计；管理员取消后，U-P05、可预约时段和统计结果应通过重新查库显示变化。
6. **完成 build/run 和权限验收。** 每轮保留实际命令、URL、账号、Cookie 行为、数据库前后状态和任务卡流转证据。

## D. 建议优化项

1. 管理端首轮只选择 T15-01、T15-02，先把认证与授权边界做稳，再开发后台业务页。
2. A-P02 使用桌面后台布局、紧凑筛选区和表格操作列；不要复用手机前台卡片布局充当后台。
3. 管理员取消应由 Service 统一校验时间、状态和 RowVersion，Controller 不直接修改 Entity。
4. 统计验收至少对比预约创建前后、管理员取消前后三组数字，避免只验证页面能打开。
5. README 当前状态准确；实际管理端功能完成后再填写可用管理员账号和演示路径。
