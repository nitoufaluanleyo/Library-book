# 《静态原型与原型评审》审计报告

## A. 总体判断

**审计结论：有条件通过。**

`prototype/static-v1/` 已完成 9 个核心静态页面，其中用户端 5 页、管理端 4 页。页面包含卡片、筛选控件、表单、表格、统计卡片、趋势区、状态标签、弹窗和滑出面板，不是“纯文本 + 链接目录页”。两个端共用 Bootstrap、本地自定义样式和统一设计变量，视觉方向基本一致。

用户端具备手机优先的单列布局、固定底部导航、44px 左右的触控区域和底部安全留白；管理端具备侧边栏、筛选区、数据表格、统计区和操作区，端侧区分清楚。9 个页面的相对资源和页面链接检查结果为 0 个断链。

当前主要问题不在页面是否完成，而在评审口径：现有自动化脚本和截图默认验收 `static-v2`，不能直接作为 `static-v1` 已完成浏览器视觉验收的完整证据；评审清单也没有单独列出“可选优化项”，且“接后端前必改项”混合了原型修正和后续开发任务。上述问题应在进入代码开发前收口。

### 已通过检查

1. `static-v1` 共 9 个 HTML 页面，数量与固定页面口径一致。
2. 用户端 5 页均有顶部区、主要内容区、操作区域和移动端底部导航。
3. 座位列表使用卡片，预约提交和管理员登录使用真实表单控件。
4. 管理端预约管理、座位管理使用侧边栏、筛选栏、表格和操作列。
5. 统计页包含 5 个数据卡片、趋势区和状态分布区。
6. 座位编辑使用同页 Offcanvas，取消预约使用 Modal，没有额外扩页。
7. 页面统一引用本地 Bootstrap、`assets/styles.css` 和 `assets/prototype.js`。
8. 主色、状态色、圆角、间距、卡片边框和按钮风格由共享样式统一控制。
9. 用户主链路与管理主链路可通过相对链接完成，静态断链数为 0。
10. 成功、空、错误、停用和冲突等状态已有静态展示入口。
11. `prototype/review-1/原型评审清单.md` 已存在，并覆盖页面、流程、状态、响应式和可访问性检查。

## C. 必改项

以下事项不要求回改本次审计对象，但必须在“系统设计说明”定稿前完成口径确认：

1. **明确正式原型基线。** 系统设计和 Razor 开发应明确采用 `static-v2` 的统一假数据与画布口径，`static-v1` 只保留为第一版结构对照，不得混用两版数字。
2. **修正验收证据表述。** 后续文档需明确：`static-v1` 已通过页面数量、结构和断链检查；当前 Playwright 截图、12 个视图和交互链路结果对应 `static-v2`。
3. **拆分评审事项类型。** 将现有第 10 节至少拆成“原型/文档必改项”和“系统设计与开发实现项”，避免把 Cookie、EF Core、防伪令牌等开发任务误判为静态页面缺陷。
4. **补齐可选优化项。** 在后续任务板或系统设计说明中单列可选优化，不得与阻塞开发的必改项混排。
5. **冻结核心数据口径。** 系统设计前固定体验用户、座位、预约、状态和统计口径，数据库实体与种子数据以该口径为准，不以 `static-v1` 页面中的零散展示数字为准。

## D. 建议优化项

1. 为 `static-v1` 增加独立的简要验收记录，至少注明测试地址、画布、页面数和断链结果；不必重复生成整套截图。
2. 在评审清单中增加“负责人、完成阶段、验收方式”三列，使问题可以直接进入任务板。
3. 将文本符号图标在 Razor 开发阶段统一替换为项目已有图标库，并继续保留可访问名称。
4. 为手机端补测 320px 最小宽度，为管理端补测约 1024px 的窄桌面宽度。
5. 系统设计时建立“原型字段 - ViewModel 字段 - 数据实体字段”对照表，减少静态页面转 Razor 时的字段遗漏。
6. 保留当前状态查询参数作为评审入口，但生产代码中不要继续使用 `prototype.js` 模拟业务结果。

## E. 是否可以进入下一步“系统设计说明”

**可以有条件进入。**

页面规模、用户主链路、管理主链路、端侧布局和统一视觉已经达到系统设计所需的原型成熟度，不需要继续扩充页面。进入下一步时必须先执行以下口径：

1. 正式视觉与数据参考版本使用 `prototype/static-v2/`，`static-v1` 仅作版本对照。
2. 系统设计围绕现有 9 页展开，不新增管理首页、确认页或独立编辑页。
3. 先冻结核心数据对象、预约状态流转、冲突规则、登录方式和统计口径。
4. 将本报告 C 节作为系统设计前置条件，将 D 节作为非阻塞优化项。

满足以上条件后，可以进入《系统设计说明》，并继续准备 ASP.NET Core MVC、Razor、EF Core 与 SQL Server LocalDB 的代码开发。
