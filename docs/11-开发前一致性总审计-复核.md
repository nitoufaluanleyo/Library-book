>

## A. 总体判断

**复核结论：不通过，退回 11a 继续修订。**

11a 的审计框架基本成立：已覆盖 docs/01-10、README 和任务板；范围、页面、数据、实现、验收五类一致性均有独立章节；21 个问题包含跨文档冲突、源文档、分级、修订状态和 Go 依据。提交 `6b577ec` 也证明直接修订真实发生，不是只更新了审计报告。

阶段边界判断正确。当前没有 `.sln`、`.csproj`、`src/` 和 `Migrations/`，README 与 docs/10 明确将其列为 Sprint 0 待生成内容，11a 没有按最终仓库形态误判开发准备文档。

但“5 个 P0 已复核、P0 剩余 0、Go”不成立。复核发现 U-P03/U-P04 页面职责、ReservationCreateViewModel 契约和验收任务映射仍有跨文档残留；静态原型的手机导航也没有达到 docs/06 和 docs/11 宣称的统一结果。任务板和 README 目前写出的“P0 已清零 / Go”因此缺少证据支撑。

## B. 发现的问题清单

### B1. P0：ReservationCreateViewModel 契约仍然冲突

| 对照位置 | 当前内容 |
| --- | --- |
| `docs/07-系统设计说明.md` | ReservationCreateViewModel 包含 SeatId、座位摘要、当前体验用户、日期/时段选项、两类占用状态和错误信息 |
| `docs/09-关键链路详细设计.md` 第 2.1 节 | 写成“ReservationCreateViewModel 只包含 SeatId、ReservationDate、TimeSlot” |

docs/09 本意应限定“POST 可提交且可被模型绑定的身份外字段”，不能把完整页面 ViewModel 缩成三个字段。该冲突直接落在 11a 标记“已复核”的 CONS-11 上，说明 U-P04 工程契约尚未清零。

### B2. P0：PRD 和数据库仍残留“时段属于座位详情页”的旧口径

1. `docs/03-PRD-Lite.md` 第 102 行仍写“具体时段可用性在座位详情页判断”。
2. `docs/03-PRD-Lite.md` 第 113、117 行仍写用户在“详情页”选择日期和时段。
3. 同文档第 221 行又明确 U-P03 只放座位详情，日期和时段属于 U-P04。
4. 同文档第 224 行写“不生成独立的座位详情”，与固定存在的 U-P03 直接矛盾。
5. `docs/08-数据库设计.md` 第 212 行仍把预约时段索引描述为“座位详情的时段占用查询”。

这些残留会让开发者把日期、时段和查询代码放回 U-P03，直接影响 View、Controller、Service 和查询职责。CONS-01、CONS-07 不能标记为“已复核”。

### B3. P0：门禁状态与实际复核结果不一致

当前以下结论需要由 11a 回退并重新复核：

1. `docs/11` 写“P0 当前剩余数量：0”和“Go”。
2. 任务板将 CONS-01、CONS-07、CONS-11 标为“已复核”，迭代记录写“P0 已复核，总审计 Go”。
3. README 写“开工前 P0 文档冲突已清零”。

在 B1、B2 修复前，上述门禁状态不能成立。

### B4. P1：CONS-05 只改了说明文档，没有与原型实物对齐

`docs/05` 使用“首页、座位、我的预约”，`docs/06` 和 `docs/11` 使用“首页、座位列表、我的预约”，而 `prototype/static-v2` 五个用户页的手机底部导航仍显示“首页、座位、预约”。三处并未统一。11a 的允许回改范围不包含原型目录，因此可以把该项留作后续修订，但不能将 CONS-05 标为“已复核”并宣称原型导航已经统一。

### B5. P1：AT01 与 M1 的任务映射不一致

`docs/11` 第 7 节写“AT01 -> T12-02 至 T12-10 / M1”；任务板中的 M1 实际为“T12-02、T12-03、T12-05 至 T12-11”。docs/11 的写法误包含 T12-04，并漏掉 M1 已纳入的 T12-11，验收映射尚未与任务板最终版本对齐。

### B6. P1：CONS-13 的延期合理，但关闭动作可更明确

仓库根 README 当前仍只有 `# Library-book`。该文件不在 11a 允许直接回改的源文档范围内，因此保留 CONS-13 合理，也不应阻塞 Sprint 0。任务板已写明 T12-10 前关闭，但 T12-10 验收标准没有显式包含“仓库根入口可发现”，建议补成独立可勾选验收点。

### B7. 分级复核

CONS-01、CONS-07、CONS-11 作为 P0 合理，当前仍有残留。CONS-17 和 CONS-19 的 P0 级别偏保守：前者主要是门禁台账完整性，后者是字段命名精度；更接近 P1 文档修订项。但两项已修正，这个分级偏差不影响本次退回结论。

其余已抽查修订项中，日期范围、5+4 页面、A-P03 同页维护、最终路由、A-P02 筛选边界、TimeProvider、统计名称、过滤索引正式字段、种子数量、Git 状态和 T12-04/T12-11 拆分均已落到对应源文档。
