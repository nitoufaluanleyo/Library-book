# 图书馆座位预约系统页面卡与 UI 规范

## 文档信息

| 项目 | 内容 |
| --- | --- |
| 文档名称 | 图书馆座位预约系统页面卡与 UI 规范 |
| 输入文件 | `docs/04-页面树与业务流程.md` |
| 当前阶段 | 页面树确认后、静态原型制作前 |
| 页面基线 | 用户端 5 页 + 管理端 4 页，共 9 页 |
| 设计优先级 | 用户端手机竖屏优先；管理端桌面后台优先 |
| 主视觉方向 | 安静学习感的校园服务界面 |

本文件不增加页面和功能。页面卡使用 `U-P01` 至 `U-P05`、`A-P01` 至 `A-P04` 编号，后续静态原型必须沿用。

### 页面卡 4：U-P04 预约提交页

| 页面卡项目 | 具体说明 |
| --- | --- |
| 1. 页面名称 | U-P04 预约提交页 |
| 2. 页面角色 | 学生体验用户 |
| 3. 页面目标 | 让用户用最少步骤确认座位、选择有效日期和固定时段，并完成一次无冲突预约 |
| 4. 页面入口 | U-P03 对启用座位点击“预约此座位”；必须携带有效 Seat Id |
| 5. 页面出口 | 提交成功进入 U-P05；取消返回 U-P03；座位不存在返回 U-P02；座位已停用返回 U-P03 |
| 6. 最重要的信息 | 座位编号和区域摘要、含当天共 7 个自然日、3 个固定时段、每个时段的可用/不可用状态、当前体验用户 |
| 7. 最重要的操作 | 选择日期；选择时段；提交预约；取消返回 |
| 8. 成功状态 | 服务端创建 1 条 Reserved 预约后跳转 U-P05，顶部显示“预约成功”；成功后刷新 U-P05 不重复提交 |
| 9. 空状态 | 所选日期没有可用时段时，3 个时段均显示不可用，并显示“当天暂无可预约时段”；仍允许切换其他日期 |
| 10. 异常提示 | 座位冲突：“该座位在所选时段已被预约”；用户冲突：“你在所选时段已有预约”；过期时段、范围外日期、停用座位和体验用户失效分别显示明确原因；体验用户失效时返回 U-P01 恢复账号 |
| 11. 手机端或桌面端布局要求 | 手机竖屏按“座位摘要 → 7 天日期横向选择 → 3 个固定时段 → 提交区”单列排列，触控项高度至少 44px，提交区不遮挡内容；桌面端表单最大宽度 720px，日期和时段保持稳定网格 |
| 12. 与后端或数据相关的关键点 | 当前用户只从 Session 读取；日期仅允许今天至今天 + 6 天；今天已开始时段不可选；写入前服务端校验座位冲突和用户冲突；重复请求必须拒绝，数据库只保留 1 条有效预约 |

## 第二部分：UI 规范与原型准备说明

### 1. 视觉方向说明

主方向明确为：**安静学习感的校园服务界面**。

用户端要像学校提供的可靠日常服务，而不是活动宣传页；管理端延续同一品牌色，但信息密度更高、装饰更少，呈现现代后台感。两端都以清晰状态、稳定布局和快速操作为优先，不使用大幅 Hero、营销文案或装饰性插画占据首屏。

设计取向：

1. 安静：大面积浅中性背景，主色只用于重点操作和当前导航。
2. 可靠：状态同时使用颜色和文字，错误信息靠近操作位置。
3. 有秩序：统一网格、间距、按钮尺寸和表格列，不随内容变化跳动。
4. 有校园感：使用克制的松柏绿作为主色，少量暖金用于提醒和学习氛围点缀。
5. 不做单一绿色界面：正文、链接、提醒和状态分别使用中性色、蓝色、金色与红色。

### 2. 色彩规范

#### 2.1 基础色

| 色彩名称 | 色值 | 使用位置 | 使用限制 |
| --- | --- | --- | --- |
| 主品牌绿 | `#1F6B5C` | 主按钮、当前导航、选中控件、关键链接 | 不作为整页大面积背景 |
| 主品牌深绿 | `#174F45` | 主按钮悬停/按下、深色强调文字 | 不用于长段正文 |
| 暖金强调 | `#D99A2B` | 少量提醒、当天标识、学习氛围点缀 | 不用于大面积按钮或背景 |
| 链接蓝 | `#2563A6` | 文本链接、可查看详情入口 | 不与主按钮争夺视觉层级 |
| 页面背景 | `#F5F7F6` | 全站页面底色 | 不叠加渐变或纹理 |
| 表面白 | `#FFFFFF` | 表格、表单、统计卡片、对话框 | 禁止卡片内再套白色卡片 |
| 主文字 | `#202725` | 标题、正文、关键数字 | 保证与白色背景对比清楚 |
| 次文字 | `#65716D` | 辅助信息、时间、备注 | 不用于关键操作和错误信息 |
| 边框灰 | `#D7DEDB` | 输入框、表格分隔、卡片边框 | 避免使用过深边框造成压迫感 |

#### 2.2 状态色

| 状态 | 文字色 | 浅背景色 | 适用对象 |
| --- | --- | --- | --- |
| 成功/启用 | `#25613B` | `#E8F4EC` | 保存成功、座位启用 |
| 已预约/信息 | `#2457A6` | `#E8F0FE` | 预约有效、普通信息反馈 |
| 警告/不可用 | `#8A5A00` | `#FFF4D8` | 时段不可用、当天提示 |
| 错误/已取消 | `#9B2C2C` | `#FDECEC` | 校验失败、取消状态、危险操作 |
| 已完成/停用 | `#58635F` | `#ECEFED` | 动态已完成、座位停用 |

状态不得只依赖颜色。标签必须同时显示“已预约、已取消、已完成、启用、停用、不可用”等文字。

### 3. 字体规范

#### 3.1 字体族

优先使用系统字体，中文顺序为：系统界面字体、Microsoft YaHei、无衬线字体。数字和英文沿用同一套系统字体，不额外引入装饰字体。

#### 3.2 字号与行高

| 文字层级 | 字号 | 行高 | 字重 | 使用位置 |
| --- | --- | --- | --- | --- |
| 页面标题 | 24px 桌面 / 22px 手机 | 32px / 30px | 600 | 每页唯一主标题 |
| 区块标题 | 18px | 26px | 600 | 待使用、历史记录、筛选区标题 |
| 组件标题 | 16px | 24px | 600 | 座位编号、卡片标题、对话框标题 |
| 正文 | 14px 桌面 / 15px 手机 | 22px / 24px | 400 | 正文、列表主要信息 |
| 表格文字 | 14px | 20px | 400 | 管理端表头与单元格 |
| 辅助文字 | 13px | 20px | 400 | 备注、时间、字段说明 |
| 按钮文字 | 14px | 20px | 600 | 主要和次要按钮 |
| 统计数字 | 28px | 36px | 600 | A-P04 数字指标 |

不使用随视口连续缩放的字号，不使用负字间距。紧凑后台也不得把正文压到 12px 以下。

### 4. 页面氛围关键词与禁用风格

#### 4.1 氛围关键词

安静、可信、克制、校园服务、学习秩序、清楚、可扫描、轻量、现代后台。

#### 4.2 禁用风格

1. 禁止紫色或蓝紫渐变作为主视觉。
2. 禁止渐变背景、玻璃拟态、发光边框、离散装饰圆球和虚化光斑。
3. 禁止大幅 Hero、宣传式口号、功能介绍长文和占据首屏的插画。
4. 禁止把每个区块都做成悬浮卡片，禁止卡片嵌套卡片。
5. 禁止大圆角和所有控件胶囊化；业务卡片圆角不得超过 8px。
6. 禁止重阴影、纯黑大面积背景、高饱和撞色和全页面单一绿色。
7. 禁止用表情符号充当图标，禁止手绘不统一的 SVG 图标。
8. 禁止只用颜色表达状态，禁止低对比度浅灰正文。
9. 禁止管理端出现营销网站式大留白和超大标题。
10. 禁止原型只放标题、纯文本和跳转链接，必须呈现真实控件、状态和样例数据。

### 5. 基础布局、间距与卡片感

#### 5.1 间距系统

统一使用 4px 基础倍数：4、8、12、16、24、32、40、48px。

1. 手机页面左右边距 16px，区块间距 24px，列表项内部间距 12px。
2. 用户端桌面内容最大宽度 1120px，左右边距至少 24px。
3. 管理端桌面侧边栏 224px，内容区内边距 24px，表格与筛选区间距 16px。
4. 表单标签与输入框间距 8px，字段组之间 16px，表单操作区上方 24px。

#### 5.2 圆角、边框与阴影

1. 按钮、输入框、筛选控件圆角 6px。
2. 统计卡片、移动端座位/预约项、登录面板圆角 8px。
3. 状态标签圆角 4px，不使用全圆胶囊。
4. 普通卡片使用 1px 边框，默认不加阴影。
5. 对话框、下拉菜单、右侧滑出面板可使用轻阴影，阴影只用于表达层级。
6. 同一页面不超过两层表面层级，禁止卡片内再放卡片。

### 6. 组件规范

#### 6.1 按钮

1. 主按钮使用主品牌绿底和白字，每个操作区只保留一个主按钮。
2. 次按钮使用白底、边框灰和主文字；危险操作使用红色文字或浅红边框，确认后才执行。
3. 手机端主要按钮高度至少 44px，桌面后台按钮高度 36 至 40px。
4. 用户端主流程按钮在手机端可全宽；管理端工具栏按钮保持紧凑，不铺满整行。
5. 返回、关闭等熟悉操作使用图标按钮并提供悬停提示；保存、预约、取消预约等明确命令使用图标加文字或文字按钮。
6. 若项目引入图标资源，统一使用 Bootstrap Icons，不再引入第二套图标系统；静态原型未引入图标库时，可使用带可访问名称的熟悉符号或文字按钮，不得为图标扩大业务依赖。
7. 提交中按钮保持原尺寸、禁用重复点击，并显示简短进行中状态，不能导致布局跳动。

#### 6.2 输入框与选择控件

1. 输入框桌面高度 40px，手机高度至少 44px，边框 1px，圆角 6px。
2. 标签固定放在控件上方，不只使用占位符代替标签。
3. 必填错误放在字段下方，使用错误色和具体文字；页面顶部可补充总体错误，但不能只有顶部错误。
4. 日期使用 7 个稳定宽度的日期选项；时段使用 3 个单选块，选中、不可用和已过期状态必须可区分。
5. 区域和状态筛选使用下拉选择；二元状态使用开关或选择控件，不使用自由文本。
6. 焦点状态使用主品牌绿边框和清晰外框，键盘操作时必须可见。

#### 6.3 表格

1. 管理端 A-P02、A-P03 桌面端使用表格，表头高度约 44px，数据行最低 48px。
2. 表头使用浅中性背景，文字 14px、字重 600；正文不使用全粗体。
3. 操作列固定在右侧并保持稳定宽度，编辑使用图标按钮，取消预约使用清楚的危险操作按钮。
4. 空状态在表格内容区居中显示，不另建空状态卡片。
5. 窄屏允许表格横向滚动，优先保留名称、状态和操作列，不强行缩小字号。
6. 不使用斑马纹堆叠颜色，以边框和轻微悬停背景辅助扫描。

#### 6.4 卡片

1. 卡片只用于 A-P04 统计指标、移动端重复座位项、移动端预约项和登录表单等确有边界的内容。
2. 卡片圆角 8px、1px 边框、白色表面，默认无重阴影。
3. 卡片内部使用固定信息层级：标题、主要数据、辅助信息、操作区。
4. 同组卡片保持一致最小高度，状态标签和操作按钮不能改变卡片尺寸。
5. 页面区块本身不套卡片，禁止卡片嵌套。

#### 6.5 弹窗与滑出面板

1. 用户取消和管理员取消使用简单确认弹窗，桌面宽度约 420px，手机端距屏幕左右 16px。
2. 弹窗只包含标题、影响说明、取消和确认操作，不放复杂表单。
3. A-P03 新增/编辑使用右侧滑出面板，桌面宽度约 420px，手机端占满可用宽度。
4. 面板关闭前若有未保存修改，应使用简短确认；关闭后仍停留 A-P03。
5. 弹窗和面板出现时焦点进入容器，关闭后返回触发按钮。

#### 6.6 状态标签与反馈

1. 状态标签使用 13px 文字、4px 圆角、浅色背景和深色文字。
2. 标签文字固定为：启用、停用、已预约、已取消、已完成、可预约、不可用。
3. 成功反馈使用页面顶部或操作区附近的短提示，自动消失前至少可读。
4. 错误反馈保留到用户修正或关闭，不自动快速消失。
5. 所有反馈同时包含图标、文字和状态色，不只显示颜色块。

#### 6.7 顶部导航

1. 用户端手机顶部栏高度 56px，显示系统短名称和当前体验账号入口。
2. 用户端手机底部导航高度 56px，固定 3 个主入口：首页、座位、我的预约；内容区预留底部空间，避免遮挡。
3. 用户端桌面使用高度 64px 的顶部导航，左侧系统名称，右侧页面入口和体验账号切换。
4. 管理员入口放在用户端次要菜单或页脚，不与学生主操作争夺位置。
5. 当前页面导航项有主品牌色和清晰选中标识。

#### 6.8 侧边栏

1. 管理端桌面侧边栏宽 224px，包含系统名称、统计、预约管理、座位管理和退出。
2. 当前页面使用浅绿背景和左侧强调线，不只改变文字颜色。
3. 侧边栏保持固定，内容区独立滚动；页面标题和工具栏位于内容区。
4. 小于 992px 时侧边栏收起为顶部菜单按钮和滑出导航，不压缩成只有难以识别的图标列。
5. A-P02、A-P03、A-P04 均显示退出入口，退出后进入 A-P01。

### 7. 响应式原则

#### 7.1 断点策略

| 范围 | 主要设备 | 设计策略 |
| --- | --- | --- |
| 小于 576px | 手机竖屏 | 用户端主验收尺寸；单列、44px 触控目标、底部导航 |
| 576px 至 991px | 大屏手机与平板 | 用户端适度双列；管理端侧边栏转抽屉；表格可横向滚动 |
| 992px 及以上 | 桌面与笔记本 | 管理端主验收尺寸；固定侧边栏、横向筛选、数据表格 |

#### 7.2 通用原则

1. 用户端先完成 390px 左右宽度的手机竖屏，再扩展桌面；管理端先完成 1366px 宽度桌面，再检查窄屏。
2. 不使用随视口连续变化的字号；通过布局换行、列数和容器宽度响应。
3. 固定格式控件使用稳定尺寸，动态状态和提示不得推挤相邻操作。
4. 任何页面不得出现文字遮挡、按钮溢出、横向页面滚动或底部导航覆盖内容。
5. 管理端表格自身可以横向滚动，但整个页面不应横向滚动。
6. 长座位编号、长体验用户名和错误提示必须测试换行，不能截断关键操作文字。

### 8. 原型三层次说明

#### 8.1 低保真原型

目标是确认页面结构和流程，不评审颜色细节。

必须表现：页面区域、信息优先级、字段和按钮位置、页面跳转、成功/空/异常状态、手机与桌面主要布局。使用灰阶框线和真实长度的样例文字，不能只画空方框。

#### 8.2 中保真原型

目标是确认组件和信息密度，作为静态 HTML 原型的直接依据。

必须应用：本文件的字号、间距、圆角、边框、按钮层级、表格列、状态标签、顶部导航、侧边栏和响应式规则。内容使用真实样例数据，例如 A-001、安静区、2026-07-14、下午 14:00-18:00。

#### 8.3 高保真原型

目标是确认最终视觉一致性和开发还原度。

必须应用：完整色彩、图标、焦点状态、悬停状态、禁用状态、成功/错误反馈、弹窗和滑出面板。高保真不增加功能，只细化已经评审通过的中保真页面。

### 9. 原型制作顺序

#### 9.1 先做低保真的页面

这些页面流程或状态较多，必须先确认结构：

1. U-P02 座位列表页：确认移动端列表、筛选和状态层级。
2. U-P03 座位详情页：确认详情与预约提交的边界。
3. U-P04 预约提交页：确认日期、时段、校验和成功跳转。
4. U-P05 我的预约页：确认待使用/历史排序和取消操作。
5. A-P02 预约管理页：确认筛选区、表格和取消处理。
6. A-P03 座位管理页：确认列表与同页新增/编辑面板，不拆页。

以上 6 页低保真通过后再升为中保真。

#### 9.2 可以直接做中保真的页面

这些页面结构简单、功能稳定，可以直接套用 UI 规范：

1. U-P01 用户首页。
2. A-P01 管理员登录页。
3. A-P04 统计页。

直接制作中保真不代表跳过流程检查，仍需提供成功、空和异常状态。

### 10. 静态原型交付要求

1. 原型必须覆盖 9 个固定页面，不多页、不少页。
2. 每页必须有真实页面框架、导航、标题、内容区、操作控件和至少一组样例数据。
3. U-P01 至 U-P05 至少提供 390px 手机竖屏视图；A-P01 至 A-P04 至少提供 1366px 桌面视图。
4. U-P04、U-P05、A-P02、A-P03 必须额外展示至少一个异常或空状态。
5. 页面之间的主按钮和返回操作必须能按页面树顺序串联。
6. 不允许使用“此处为列表”“点击后跳转”等说明文字代替真实列表、按钮和状态。
7. 不允许因为原型工具方便而新建独立成功页、确认页、新增页或编辑页。

### 11. 原型评审重点

#### 11.1 业务与页面范围

1. 是否严格保持用户端 5 页、管理端 4 页。
2. 用户能否从 U-P01 连续完成查座、详情、预约、查看和取消。
3. 管理员能否从 A-P01 连续完成登录、预约处理、座位维护、统计和退出。
4. 是否错误增加注册、预约结果、座位新增/编辑、管理首页等页面。

#### 11.2 信息与操作

1. 每页最重要的信息是否在首屏或主要内容区可见。
2. 每个操作区是否只有一个明确主操作，危险操作是否有确认。
3. 座位启停和时段可用性是否被正确区分。
4. 已预约、已取消、已完成、启用、停用是否同时使用文字和颜色。
5. 成功、空、异常和无权限状态是否都有去向，不出现死路。

#### 11.3 布局与视觉

1. 用户端 390px 手机竖屏是否无需横向滚动，按钮是否便于触控。
2. 管理端 1366px 桌面是否能完整显示筛选区、关键表格列和操作区。
3. 卡片是否克制，是否出现卡片嵌套、重阴影、大圆角或营销式 Hero。
4. 色彩是否符合安静学习感，状态对比是否清楚，页面是否过度单一绿色。
5. 字号、间距、圆角、按钮高度和表格行高是否按规范统一。
6. 长文字、错误提示和状态变化是否造成溢出、遮挡或布局跳动。

原型评审通过后，下一步进入《静态原型与原型评审》，先完成核心 6 页的低保真流程评审，再补齐 9 页中保真静态原型。
