## 10. 哪些问题必须在接后端前修改

以下事项必须在 ASP.NET Core MVC 接入数据库和后端前完成：

- [ ] 将 9 页公共头部、用户导航、管理侧边栏和反馈组件拆为 Razor Layout/Partial，避免复制粘贴。
- [ ] 删除或隔离 `prototype.js` 的假跳转，不把原型状态脚本当作业务逻辑。
- [ ] 将静态假数据替换为 ViewModel 和 EF Core 查询结果。
- [ ] 实现体验用户 Session，并从服务端确定当前用户，禁止前端提交任意用户编号。
- [ ] 实现管理员 Cookie 登录、退出和管理端授权拦截。
- [ ] 为所有 POST 操作增加服务端验证和防伪令牌。
- [ ] 实现日期范围、已开始时段、停用座位、座位冲突和用户冲突校验。
- [ ] 实现用户取消、管理员取消、取消方和取消时间持久化。
- [ ] 实现 StoredStatus 与动态 DisplayStatus 的统一计算方法。
- [ ] 实现座位编号唯一校验，并保持 A-P03 同页新增/编辑体验。
- [ ] 用真实统计查询替换静态数字、趋势和状态分布；若不实现趋势数据，则保留状态分布或按最终范围删减。
- [ ] 将成功、空、错误和无权限状态转为真实服务器结果。
- [ ] 复核 LocalDB 初始化、相对日期种子数据和 README 运行步骤。
